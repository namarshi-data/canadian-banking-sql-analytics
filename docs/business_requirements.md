# Business Requirements

## Objective

Create a SQL analytics layer that allows a Canadian banking analytics team to answer performance, risk, fraud, service, and customer questions from raw operational data.

## Stakeholders

- Finance leadership: deposit balance, fee income, interest income, branch target attainment.
- Risk management: customer risk segments, delinquency buckets, utilization risk, loan late payments.
- Fraud operations: alert volume, confirmed fraud rate, estimated losses, false positives.
- Branch operations: SLA performance, service request pressure, new account targets.
- Marketing: campaign conversion, channel performance, estimated revenue, targeted vs control comparison.
- BI/reporting: reusable views, documentation, and quality checks.

## Key KPIs

| KPI | Business meaning |
|---|---|
| Ending balance | Month-end balance used for portfolio size and trend reporting |
| Interest amount | Monthly interest generated by accounts/products |
| Fee amount | Monthly service/credit/product fees |
| Deposit target attainment | Actual deposit balance divided by branch target |
| Loan origination target attainment | Actual originations divided by monthly target |
| New account target attainment | New accounts opened divided by branch target |
| SLA breach rate | Share of service requests missing target resolution time |
| Confirmed fraud rate | Confirmed fraud alerts divided by total alerts |
| Conversion rate | Campaign conversions divided by contacts |
| Delinquency bucket | Late payment severity based on days late |

## Reporting grain

- Monthly institution/product KPIs: one row per month, institution, product family/category.
- Branch target attainment: one row per branch and target month.
- Customer 360: one row per customer.
- Fraud operations: one row per month, institution, alert type, severity, and status.
- Campaign performance: one row per campaign, test group, and contact channel.
- Service SLA: one row per month, institution, request type, priority, channel, and status.
