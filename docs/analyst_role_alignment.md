# Analyst Role Alignment

## Target roles

- Data Analyst
- BI Analyst
- Reporting Analyst
- Financial Data Analyst
- Banking Analyst
- Risk Analyst
- Operations Analyst
- Junior Analytics Engineer

## Job-signal mapping

| Common job requirement | How this project demonstrates it |
|---|---|
| SQL querying and database analysis | 20+ analyst SQL questions, advanced CTE/window/function examples |
| Power BI / dashboard readiness | Reusable views and materialized views designed as semantic reporting tables |
| Data quality and reconciliation | Quality test scripts, row-count reconciliation, exception extracts |
| Financial analysis | Deposit balances, fee income, interest income, branch targets, loan originations |
| Risk analysis | Customer risk history, card utilization, days past due, loan delinquency buckets |
| Fraud/AML-style operations thinking | Fraud alert severity, confirmed fraud rate, loss estimation, false-positive pressure |
| Stakeholder communication | Executive insights report and portfolio storyboard |
| Documentation | Data dictionary, ERD, business requirements, concept coverage |
| Deployment readiness | Docker Compose, Makefile, repeatable PostgreSQL load |

## Resume bullets

- Built a deployment-ready PostgreSQL analytics warehouse using 514K+ rows of synthetic Canadian banking data across customers, accounts, transactions, balances, loans, fraud alerts, campaigns, and service requests.
- Designed typed schemas, constraints, indexes, data-quality checks, reporting views, and materialized views to support finance, risk, branch, fraud, and operations analytics.
- Wrote advanced SQL case studies using CTEs, window functions, cohort analysis, ranking, Pareto analysis, rolling trends, and KPI reconciliation to answer business questions for Canadian banking stakeholders.
- Developed reusable marts for customer 360, branch target attainment, monthly institution KPIs, loan delinquency, fraud operations, campaign performance, and service SLA reporting.
